# EmailJS Security Configuration

## Защита от несанкционированного использования ключей

### 1. Ограничение доменов в EmailJS Dashboard
1. Войдите в [EmailJS Dashboard](https://dashboard.emailjs.com/)
2. Перейдите в **Settings** → **Security**
3. В разделе **Allowed Domains** добавьте только ваши домены:
   - `yourdomain.com`
   - `www.yourdomain.com`
   - `localhost` (для разработки)
4. Сохраните изменения

### 2. Настройка Rate Limiting
1. В EmailJS Dashboard → **Settings** → **Security**
2. Включите **Rate Limiting**
3. Установите лимиты (например: 100 писем в час)
4. Это предотвратит массовую рассылку с ваших ключей

### 3. Мониторинг использования
1. Регулярно проверяйте **Dashboard** → **Logs**
2. Отслеживайте необычную активность
3. Если обнаружено злоупотребление - немедленно перегенерируйте ключи

### 4. Перегенерация ключей
Если ключи скомпрометированы:
1. EmailJS Dashboard → **Settings** → **General**
2. Нажмите **Regenerate Public Key**
3. Обновите `config.js` с новым ключом

### 5. Дополнительная защита в коде
В `config.js` можно указать разрешенные домены:
```javascript
ALLOWED_DOMAINS: ['yourdomain.com', 'www.yourdomain.com']
```

**Важно:** Ограничение доменов в EmailJS Dashboard более надежно, чем проверка в коде, так как код можно обойти.

### 6. Альтернативное решение (более безопасное)
Для максимальной безопасности используйте бэкенд:
- Ключи хранятся только на сервере
- Форма отправляет данные на ваш API
- API отправляет письма через EmailJS или другой сервис
